# Persona: James Okafor — Educator

## Demographics

- **Age:** 45
- **Occupation:** Community college instructor (Introduction to Computer Science)
- **Location:** Atlanta, Georgia, USA
- **Technical comfort:** Moderate — uses GitHub for distributing starter code to students, comfortable with markdown, but prefers GUI tools over the terminal

## Background

James teaches intro CS classes and is tired of the limitations of his institution's LMS (learning management system). He wants a lightweight way to distribute supplementary course material that students can use offline — especially important for students with unreliable internet. He creates courses as GitHub repos and shares the URL with his class. Students download Course Imports and load the material.

## Goals

- Distribute structured course material to 30+ students reliably
- Include quizzes that give students instant self-assessment feedback
- Ensure the experience works offline once a course is loaded
- Keep the technology barrier low so students aren't stuck on setup

## Typical workflows

1. Prepares course material as a GitHub repository during the semester
2. Shares the repo URL on the class syllabus or LMS
3. Walks students through installing Course Imports and loading the course
4. Students work through topics and quizzes at their own pace outside class
5. Updates the course repo mid-semester and tells students to reload
6. Uses quiz blocks to create self-check exercises for each topic

## Pain points and frustrations

- Needs the app to work on both macOS and Windows since students use both
- Worries about students hitting GitHub API rate limits if 30 people load the same course at once
- Frustrated if students can't figure out how to load a course — the home screen needs to be self-explanatory
- Would like some way for students to reset their progress if they want to redo quizzes

## What success looks like

James shares a GitHub URL, all 30 students successfully load the course within the first week, work through topics and quizzes at their own pace, and nobody emails him for tech support.
